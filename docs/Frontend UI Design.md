# **03\. フロントエンド・UI設計**

## **1\. 共通UXガイドライン (Human-Centric)**

* **Philosophy:** **「システムは判断しない。判断材料を提示するだけ。」**  
* **Overridable:** エラーや警告が出ても、人間の意思で「強制実行（Override）」できる逃げ道を作る。  
* **Target:** スマホ (iPhone SE2 \~) / 縦画面専用  
* **Touch Target:** **44px以上 (親指サイズ)** を厳守。

## **2\. アプリ別要件**

### **A. Staff App (POS & Ticket)**

**「ノールック操作」と「現場裁量」**

* **Login Flow:** 「店舗招待QR」をカメラでスキャンすると、自動的にその店舗のスタッフとしてログインし、ホーム画面へ遷移する。  
* **Scan Lock:** リストバンドをスキャンしない限り、会計ボタンをDisabledにする物理制御。  
* **Wake Lock:** navigator.wakeLock を使用し、スリープを防止する。  
* **Guest Mode:** ロゴ長押しで「QRなし会計モード」へ移行する隠しコマンド。

#### **Ticket Manager (整理券管理画面)**

* **Mode Toggle:**  
  * 画面上部に巨大なトグルスイッチ。  
  * \[🟢 通常入場 (Walk-in)\] ⇔ \[🔴 整理券発券中 (Ticket Only)\]  
  * 店員がこれを切り替えない限り、勝手にモードは変わらない。  
* **Check-in & Override:**  
  * 整理券QRをスキャンした際の結果表示：  
    * 有効：画面全体が緑色になり「OK (13:00回)」と表示。  
    * 無効：画面全体が赤色になり「⚠️ 時間外 / 無効」と表示。  
  * **強制入場ボタン:** 赤画面の下に\*\*「強制入場させる (Force Check-in)」\*\*ボタンを配置し、現場判断で通せるようにする。

### **B. Visitor App (MyPage)**

**「情報ハブ」**

* **A2HS誘導:** 初回アクセス時に「ホーム画面に追加」を促すモーダルを表示。  
* **Map:** 軽量な画像/SVGベースのマップ。ピンをタップすると混雑状況が表示される。  
* **QR Brightness:** QR表示時は周囲を白く縁取り、読み取り精度を確保する。

#### **Ticket Wallet (チケット画面)**

* **Status Card:**  
  * 現在保有中の整理券を表示。「あと5組」「13:00〜13:30」など。  
  * **「遅れると無効になる場合があります」** という注釈を入れ、早めの来店を促す。  
* **Call Notification:**  
  * 自分の順番が近づいたら（または時間枠になったら）、アプリ内で目立つアラートを表示。

### **C. Stage Commander (Admin)**

**「コックピット」**

* **Trigger Button:** 演出やテロップは、**人間がボタンを押した瞬間**にのみ発動する。「タイマーで自動発動」は実装しない。  
* **Safety:** 「緊急テロップ」などの危険なボタンは\*\*「長押し(1.5s)で発火」\*\*とし、誤爆を防ぐ。  
* **Status:** 現在のOBSシーン、配信状態をリアルタイム表示する。
